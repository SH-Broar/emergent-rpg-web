# 던전 시스템 개편 (2026-04-11)

## 변경 요약

기존 방 선택(좌/우/샛길) 방식에서 **층·진행도 기반 선택지 시스템**으로 전면 개편.

### 핵심 구조

```
던전 = [층1] → (중간보스?) → [층2] → ... → [최종보스]
 층  = [단계1] → [단계2] → ... → [단계N]
단계 = N개 선택지 중 K개 클리어 → 다음으로
```

### 데이터 모델 (`DungeonDef`)

| 필드 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `floors` | int | 난이도 기반 2~6 | 총 층 수 |
| `progressSteps` | int | 난이도 기반 2~3 | 층당 진행 단계 수 |
| `choicesPerStep` | int | 5 (고난이도 6) | 단계당 선택지 N개 |
| `requiredClears` | int | N-1 | 필수 클리어 수 K |
| `midBosses` | MidBossDef[] | [] | 중간 보스 목록 |
| `floorDefs` | DungeonFloorDef[] | [] | 층별 몬스터 오버라이드 |

### 난이도별 기본값

| 난이도 | 층수 | 단계/층 | 선택지(N) | 필수(K) |
|--------|------|---------|-----------|---------|
| ~0.15 | 2 | 2 | 5 | 4 |
| 0.16~0.25 | 3 | 2 | 5 | 4 |
| 0.26~0.40 | 3 | 3 | 5 | 4 |
| 0.41~0.60 | 4 | 3 | 5 | 4 |
| 0.61~0.80 | 5 | 3 | 6 | 5 |
| 0.81~ | 6 | 3 | 6 | 5 |

### UI 흐름

1. **선택지 화면**: N개 버튼(전투/이벤트/휴식). 클리어한 선택지는 비활성 표시.
2. **K개 클리어** → "다음으로" 버튼 출현 → 다음 단계 또는 다음 층.
3. **하단 고정 버튼**:
   - 탐색: 잔여 미클리어 수 × 10% HP 소모, 즉시 진행
   - 던전 지도: 전체 층 구조 표시
   - 포기: 확인 후 진행도 미저장 퇴장

### 진행도 저장 정책

- **보스 클리어 시에만** 진행도(`pendingProgress`) 확정 저장
- 중도 포기 시 진행도, 최단 턴 기록 미저장

### 몬스터 밸런스

- 전 몬스터 attack/defense/hp × 1.5~2배 상향
- 플레이어가 선택지를 고를 수 있으므로 기존보다 높은 스펙 필요

## 추후 개선 방향

### 단기

- [ ] **층별 몬스터 풀 분리**: `floor_N_enemies` 필드로 층마다 다른 몬스터 배치. 현재 코드는 지원하지만 데이터 미적용.
- [ ] **전용 이벤트 모드**: 특정 이벤트가 발생 시 모든 선택지 대신 이벤트 전용 UI만 표시하는 `exclusive` 플래그.
- [ ] **넓은 던전 특성 강화**: 리아그랄타 등 넓은 필드 던전의 N을 7로 설정했으나, 추가 특성(발견 보너스, 스카우트 이벤트 등) 필요.
- [ ] **DeepFog 규칙 적용**: 안개 던전에서 일부 선택지를 "???" 표시로 가리는 기능.

### 중기

- [ ] **중간 보스 다양화**: 현재 기존 몬스터 재사용. 전용 중간 보스 몬스터 + 특수 기믹 추가.
- [ ] **단계별 난이도 곡선**: 같은 층 안에서도 후반 단계일수록 전투 비중 증가.
- [ ] **보스 패턴 시스템**: 최종 보스에 페이즈 전환, 특수 기믹 도입.
- [ ] **클리어 보상 차등화**: 선택지 전부 클리어(K=N) 시 추가 보상.

### 장기

- [ ] **던전 이벤트 스토리라인**: 층 진행에 따라 연결되는 이벤트 체인.
- [ ] **멀티 루트 던전**: 층 분기 — 어느 루트를 타느냐에 따라 다른 보스/보상.
- [ ] **시간 압박 던전**: 실시간 타이머 기반 던전.
- [ ] **협동 던전**: 파티원과 역할 분담 (탱커 전면/딜러 후방 등).

### 밸런스 모니터링

- 선택지 N=5, K=4가 적절한지 (너무 쉽거나 너무 많은 전투 발생)
- 탐색 HP 비용(잔여 × 10%)이 균형잡힌지
- 중간 보스 등장 빈도와 난이도 적절성
- 층당 진행 단계 수(2~3)가 플레이 리듬에 맞는지
